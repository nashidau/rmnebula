Meta Information

<!DOCTYPE rmsscharacter SYSTEM 
        "http://www.nash.nu/Rolemaster/Nebula/xml/rmsscharacter.dtd">

Basic information on the class. Aside from the xml decleration (required in all XML documents) no other element is required. The DTD decleration is strongly recommended, as is the CSS.

Root Element

<rmsscharacter id="neb-0001">

The root element. The id attribute is a global identifier, and is thus unique for every charactter. The method of allocating these is up to the individual program.

Data Section

<data>
        <name>Evant</name>

This section defines basic information about the character. In addition it hosts a lot of derived information, as well as basic background information. The first required attribute is name which contains the characters name. The other fields are:

  • player
  • campaign
  • fixed
  • experience
  • defensive
  • resistances
  • raceinfo
  • appearance
  • role
  • background

Player

        <player>
                <name>Nash</name>
                <email>nash@nash.nu</email>
                <application>Nebula</application>
        </player>

Player information. The only required field is the name field. Email if entered should be a valid edmail address, without < or >. The application is simple what program created the file.
